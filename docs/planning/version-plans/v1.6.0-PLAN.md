# Version 1.6.0 Release Plan - AI Intelligence & Creativity

## Overview

**Release Codename:** "Creative Genius"
**Target Release Date:** 2025-12-12 (Accelerated)
**Theme:** AI-Powered Intelligence & Creative Features
**Dependencies:** v1.5.0 (Accessibility & Global Reach)

**IMPORTANT: Client-Side Architecture Constraint**
This release is constrained by InfoGraphix's client-side architecture (React app deployed to Google AI Studio with NO backend infrastructure). All features must be implementable using client-side JavaScript, browser APIs, and Google Gemini AI APIs.

This release leverages advanced AI capabilities to provide intelligent suggestions, an enhanced template library, and data visualization support. The focus is on enhancing creativity and reducing cognitive load through smart automation.

---

## Revised Scope for v1.6.0

Based on the client-side architecture constraint, the following features are **feasible** for this release:

### CORE FEATURES (Must Have)
1. ✅ **AI-Powered Style Suggestions** - Use Gemini to analyze topics and recommend styles/palettes
2. ✅ **Smart Palette Generator** - Client-side color extraction from images using Vibrant.js
3. ✅ **Enhanced Template Library** - Expand to 50+ templates with categories (static, no backend)
4. ✅ **Layout Variants** - Generate 2-3 layout options using Gemini variations

### EXTENDED FEATURES (Should Have)
5. ✅ **Data Visualization Templates** - CSV import and data-aware infographic generation

### DEFERRED FEATURES (Requires Backend or Different AI Model)
6. ❌ **Animation Support** - Deferred to v1.7.0+ (Gemini Image API only produces static images)
7. ❌ **Community Template Sharing** - Deferred (requires backend database)
8. ❌ **Template Rating System** - Deferred (requires backend)

### OPTIONAL (If Time Permits)
9. ⚠️ **Presentation Mode** - Basic fullscreen presentation (low priority)

### Feature 6: Animation Support ❌ DEFERRED

**Priority:** Low
**Effort:** Very High
**Status:** DEFERRED to v1.7.0+

**Reason for Deferral:**
Gemini Image API (`gemini-3-pro-image-preview`) only generates static PNG images. Animation support requires:
- Backend video encoding infrastructure (FFmpeg), OR
- Frame-by-frame static image generation + client-side stitching (very slow, large files), OR
- New Gemini animation API capability (doesn't exist yet)

**Future Considerations:**
- Wait for Gemini video generation API
- Consider server-side deployment for video encoding
- Explore CSS animation overlays on static images (limited capability)
